# Automation and Scheduled Execution
This document explains how the data collection and pushing pipeline is automated for this project. The system is designed to:
1. Scrape municipal residential water rate data
2. Save the results to a CSV
3. Push the created CSVs to a GitHub repository
4. Optionally overwrite the hosted table in ArcGIS Online
These tasks are executed in sequence through a Python script and scheduled to run automatically using Windows Task Scheduler.

---

## Overview of the Pipeline
The automation process works in the following order:
1. ```scrape_water_rates_data.py```
   Scrapes municipal residential water rate data and creates/updates the local CSVs
2. ```push_csv_to_github.py```
   Commits and pushes the updated CSV files to a GitHub repository
3. ```push_csv_to_agol.py```
   Overwrites the hosted table in ArcGIS Online with the updated CSV

These scripts are executed sequentially by ```run_sequence.py```

Windows Task Scheduler runs **only** the ```run_sequency.py```, which ensures the steps occur in the correct order.

---

## Scripts Overview

```scrape_water_rates_data.py```: see ```docs/scraper_script_configuration.md```

```push_csv_to_github.py```
Purpose: Commit and push CSV files generated by ```scrape_water_rates_data.py``` to a GitHub repository

The script will search for the CSV files, add them to the Git staging area, commit the changes, and push the commit to the remote repository. A sample of this script is available as ```scripts/push_csv_to_github_sample.py```. This helps ensure data is stored in more than one location and not in two local locations.

```push_csv_to_agol.py```
Purpose: Update the hosted table in ArcGIS Online

This script will:
- Connect to ArcGIS Online using the ArcGIS Python API
- Authenticates with a user account
- Overwrites the hosted table with the updated main CSV

Requirements:
- ArcGIS Online account with permissions to remotely overwrite the hosted table
- Correct item ID for the hosted table
- ArcGIS Python API installed via pip (not included in the requirements.txt, use pip install arcgis in your virtual environment)

If you do not have the necessary permissions or face Dual-Factor Authentication (DFA) restrictions, skip implimenting this script

---

## Sequence.py

The ```run_sequence.py``` script ensures that the scraper script runs first, followed by ```push_csv_to_github.py``` and ```push_csv_to_agol.py```. While scripts could be sequenced in Windows Task Scheduler, this script utilizes subprocesses to ensure the data that is pushed is the most up to date and no new water rate data are missed.
